# 서울 아파트 실거래가 기반 가격 예측 프로젝트

데이터마이닝 A분반 2팀 · 학기말 프로젝트

서울 아파트 매매 실거래가에 입지(지하철·학교)·건물 특성·거시경제(기준금리) 변수를 결합하여
가격 결정 요인을 분석하고, 머신러닝 회귀 모델로 거래금액을 예측한다.

## 분석 개요

- **분석 기간**: 2023년 1월 ~ 2025년 12월 (36개월)
- **데이터 규모**: 142,463건 × 33개 변수 (전처리 후, 결측치 0건)
- **목표**: ① 가격 결정 요인 분석 ② 거래금액 예측 ③ 유사 아파트 추천

> **분석 기간 관련 메모**: 초기 계획은 2022~2024년이었으나, 국토교통부 실거래가 시스템이
> 2022년 거래의 동(棟) 정보를 공개하지 않아 "동 없는 행 제거" 전처리 규칙상 2022년이 전부 탈락했다.
> 대신 신고가 완결된 2025년을 추가하여 동일하게 36개월·금리 변동 사이클을 포함하도록 조정했다.

## 1. 데이터 수집·변수 생성 (`01_data_collection_pipeline.ipynb`)

### 1-1. 지오코딩 (주소 → 좌표)
- 실거래 데이터는 거래 1건 = 1행 구조라 같은 단지가 수십~수백 번 반복됨
- **단지명 + 도로명 주소 기준으로 고유 단지만 추출**해 한 번씩만 좌표 변환 → API 호출 약 **16배 절감**
- 변환된 좌표를 다시 원본 거래 데이터에 `left join`하여 모든 거래 행에 위치 정보 부여
- **API 전환**: 카카오 API는 테스트 앱 상태에서 일괄 호출 한도 문제 발생 → 일일 한도가 충분한
  **VWorld API(국토교통부)** 로 전환. 도로명 주소 변환 성공률 99.7%

### 1-2. 거리 변수 (haversine)
- 위·경도는 지구 곡면 좌표이므로 단순 유클리드 거리 대신 **haversine 공식**으로 실제 지리 거리 계산
- **지하철**: 최근접 역명·호선·거리(m). 서울 외곽 단지는 인접 경기·인천 역이 더 가까울 수 있어
  **수도권 전체 역을 후보로 두고** 최솟값 산출
- **학교**: 초/중/고 학교급을 분리하여 각각 최근접 학교까지 거리 + 반경 1km 내 학교 수

### 1-3. 기준금리 결합
- 한국은행 기준금리(월별 가로형) → 세로형 변환 → `계약년월` 기준 매핑

## 2. 전처리 (`02_preprocessing.ipynb`)

원본 약 17.7만 건 → 필터링·파생변수 생성 후 **142,463건**.

### 2-1. 행 필터링
| 단계 | 처리 | 근거 |
|---|---|---|
| 계약 해제 제거 | `해제사유발생일`이 있는 거래 삭제 | 취소된 거래는 실제 체결가 아님 |
| 지하층 제거 | `층 < 0` 삭제 | 주거 일반 거래로 보기 어려움 |
| 괄호+숫자 단지명 제거 | `(123)` 형태 단지 삭제 | 단지명 불명 데이터 |
| 동 결측 제거 | `동`이 숫자 형식 아닌 행 삭제 | 동 정보 없는 거래 제외 |
| 좌표 결측 제거 | 지오코딩 실패 단지 거래 삭제 | 거리 변수 산출 불가 |
| 10평 미만 제거 | `전용면적_평 < 10` 삭제 | 비주거·특수 매물 제외 |

### 2-2. 파생 변수 생성
- **거래금액 정수 변환**: `"175,000"` (콤마 포함 문자열) → `175000` (정수)
- **전용면적(평)**: `전용면적(㎡) / 3.3058`
- **평형대**: 10평 미만 / 10~20 / 20~30 / 30~40 / 40평 이상 구간화
- **건물연식**: `계약연도 − 건축년도`, 연식 구분(신축/준신축/중간/구축)
- **평당가**: `거래금액 / 전용면적(평)`
- **집계 변수**: 구별·단지별 평균 평당가
- 자치구명 추출, 주소 정제, 가나다순 정렬

## 3. 탐색적 데이터 분석 (`03_EDA_modeling.ipynb`)

### 3-1. 거래금액 분포
- 평균 **12.67억** vs 중앙값 **10.4억** → 평균 > 중앙값, **우측 꼬리 분포**(왜도 3.63)
- 일부 초고가 거래가 평균을 끌어올리는 구조 → 모델링 시 **종속변수 로그 변환** 근거

### 3-2. 입지(자치구) — 가장 강한 요인
- 평당가 상위: **서초구 9,975 · 강남구 9,973 · 용산구 7,867** (만원)
- 평당가 하위: **도봉구 2,679 · 강북구 3,026 · 금천구 3,072**
- 최상위(서초) vs 최하위(도봉) **약 3.7배** → 입지가 가격을 설명하는 핵심 변수

### 3-3. 지하철 접근성 (역세권 효과)
| 역 거리 | 평균 평당가(만원) |
|---|---|
| 250m 이내 | 5,993 |
| 500m 이내 | 5,643 |
| 1km 이내 | 4,995 |
| 2km 이내 | 3,883 |
| 2km 초과 | 3,632 |

→ 역과 가까울수록 평당가가 단조 증가 (역세권 프리미엄 정량 확인)

### 3-4. 건물 연식 — 비선형 관계
- 평당가: 준신축(6,835) > 신축(6,295) > 중간(5,698) > 구축(4,742)
- **신축이 가장 비쌀 것이라는 통념과 달리 준신축이 최고가** → 핵심 입지 단지 비중 영향
- 구축도 재건축 기대로 가격을 유지하는 경우 존재 → **선형 회귀로는 포착 어려움 → 트리 모델 필요**

### 3-5. 상관관계 분석 및 변수 선택
- 거래금액과 강한 상관: 평당가 계열, 전용면적
- 역 거리·층·학교 변수는 단독 상관은 낮으나 **보조 변수**로 활용 가능
- `건축년도`와 `건물연식`은 거의 완전한 역상관(다중공선성) → **건물연식 하나만 사용**
- **핵심 변수(입지·면적)와 보조 변수를 구분**하여 모델 입력 변수 설계

## 4. 모델링 (`03_EDA_modeling.ipynb`)

타깃 `거래금액` 로그 변환, 학습:테스트 = 8:2.

### 4-1. 5개 모델 비교

| 모델 | RMSE(만원) | R² |
|---|---:|---:|
| 선형 회귀 | 45,012 | 0.8806 |
| K-최근접 이웃 | 18,014 | 0.9605 |
| 랜덤 포레스트 | 12,994 | 0.9788 |
| 그래디언트 부스팅 | 23,212 | 0.9414 |
| **히스토그램 기반 부스팅** (채택) | 18,511 | 0.9647 |

- **선형 회귀**: RMSE 최대 → 비선형 구조 포착 못 함 (EDA의 연식 비선형성과 일치)
- **랜덤 포레스트**: 성능 1위지만 학습·예측 속도 느림
- **히스토그램 기반 부스팅 채택**: 성능과 처리 속도의 균형이 우수

### 4-2. 변수 중요도 → 최종 6개 입력 변수
permutation importance로 13개 → 중요도 0.01 이상 **6개** 선택:

| 순위 | 변수 | 중요도 |
|---|---|---:|
| 1 | 단지별 평균평당가 | 0.5265 |
| 2 | 전용면적(평) | 0.3337 |
| 3 | 구별 평균평당가 | 0.2762 |
| 4 | **기준금리** | 0.0364 |
| 5 | 건물연식 | 0.0328 |
| 6 | 최근접 역 거리 | 0.0101 |

> **주목**: 기준금리는 단독 상관계수는 미미(약 -0.06)했으나 트리 모델 변수 중요도에서는 **4위**로
> 진입 — 선형 상관으로는 잡히지 않지만 다른 변수와의 비선형·상호작용으로 가격에 작용함을 보여준다.
>
> **데이터 누수 주의**: 평당가(`거래금액/면적`)는 타깃 정보를 직접 포함하므로 입력 변수에서 제외하고,
> 지역성·단지 특성을 요약하는 단지별·구별 평균 평당가만 사용했다.

### 4-3. 최종 성능
- **MAE 10,173만원** (평균 거래금액 12.7억 대비 약 8%)
- **RMSE 18,971만원** — MAE보다 큰 격차는 일부 고가·특수 입지 거래에서 큰 오차 발생을 의미

### 4-4. 유사 아파트 추천
- `NearestNeighbors`로 6개 변수 기준 입력 조건과 가장 가까운 **5개 단지** 제시
- 단순 예측가뿐 아니라 실제 유사 거래 사례를 함께 제공 → 예측 결과의 해석 가능성 보완
